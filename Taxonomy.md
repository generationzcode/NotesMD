## Objects and Features
- an object is a observation, an entity, a training instance. It is basically a row of data
- a feature is a column of data, its a part of an object. Black hair would be a feature, the object could be a person
- The object language is the formalism in which objects and features can be described
### Features
- they can be ordinal (binary), discrete numerical, continuous numerical, symbolic, graphs, lists
### Object Space
- The population of the objects
- The data you will have will always be a subset of the object space, since the object space is like a vector space (note how I didn't say it is exactly) that has all possible objects
- The extension of a concept of, say, a buffalo from data can be the set of all buffaloes in real life
## Category and Hypothesis
- Category is a concept of a class
- Hypothesis is the intention of a concept, the model used to classify
- The hypothesis(Concept,Category) language can be theoretically different to the object language but is syntactically the same
- The generalization of knowledge from an object goes to category definition. The object is an instance of the category definition
- The object is an element of the category itself, which is the subset of everything in the dataset consistent with the category definition
- fish 0,0,?,<3 could be a category definition
## Taxonomy
- specialization is excluding a negative instance, generalization is including a positive instance
